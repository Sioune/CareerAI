## 1. 接入准备

### 1.1 获取 API Key 和 API Secret

联系支付服务管理员，为您的业务系统创建 API Key。创建后您将获得：

| 参数 | 说明 |
|------|------|
| `apiKey` | 身份标识，用于请求头 `X-App-Id` |
| `apiSecret` | 签名密钥，用于计算 HMAC-SHA256 签名（**仅在创建时展示一次，请妥善保管**） |

### 1.2 通用约定

| 约定 | 说明 |
|------|------|
| 通信协议 | HTTPS |
| 数据格式 | JSON（`Content-Type: application/json`） |
| 字符编码 | UTF-8 |
| 时间标准 | Unix 时间戳（秒级） |
| 金额单位 | **分**（如 `100` 表示 1 元） |

## 2. 签名机制

所有业务接口均需通过签名验证。每个请求必须携带以下 **5 个自定义请求头**：

| 请求头 | 说明 | 示例 |
|--------|------|------|
| `X-App-Id` | 您的 API Key | `ak_abc123` |
| `X-Timestamp` | 当前 Unix 时间戳（秒） | `1712736000` |
| `X-Nonce` | 随机唯一字符串，防重放 | `uuid-or-random-string` |
| `X-Body-Digest` | 请求体的 SHA-256 摘要（小写 hex） | `e3b0c44298fc...` |
| `X-Sign` | 请求签名（HMAC-SHA256） | `a1b2c3d4e5f6...` |

### 2.1 签名计算步骤

#### 第一步：计算请求体摘要（Body Digest）

对请求体的原始字节计算 SHA-256，结果转为**小写十六进制**字符串。

```
bodyDigest = SHA-256(requestBody).toLowerCaseHex()
```

- 对于 GET 请求（无请求体），对空字符串计算：`SHA-256("")` = `e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855`
- 对于 POST 请求，对完整的 JSON 字符串原始字节计算

#### 第二步：构造待签名字符串

将以下参数放入 Map 中，按**键名字典序**排列，然后用 `&` 拼接为 `key=value` 格式：

| 参数名 | 取值来源 |
|--------|----------|
| `appId` | `X-App-Id` 请求头的值 |
| `bodyDigest` | 第一步计算的 Body Digest |
| `nonce` | `X-Nonce` 请求头的值 |
| `timestamp` | `X-Timestamp` 请求头的值 |
| *（如有 URL Query 参数）* | 查询参数也参与签名 |

拼接结果示例（无 Query 参数时）：

```
appId=ak_abc123&bodyDigest=e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855&nonce=my-nonce-123&timestamp=1712736000
```

#### 第三步：计算 HMAC-SHA256 签名

使用 `apiSecret` 作为密钥，对**待签名字符串**计算 HMAC-SHA256，结果转为**小写十六进制**字符串。

```
signature = HMAC-SHA256(stringToSign, apiSecret).toLowerCaseHex()
```

### 2.2 安全机制说明

| 机制 | 说明 |
|------|------|
| 时间戳校验 | 请求时间戳与服务端时间差不得超过 **300 秒** |
| Nonce 防重放 | 每个 Nonce 在 300 秒内不可重复使用 |
| Body Digest | 确保请求体在传输过程中未被篡改 |

### 2.3 签名流程图

```
请求体 → SHA-256 → bodyDigest
                          ↓
appId + bodyDigest + nonce + timestamp + [queryParams]
                          ↓
              按键名字典序排列，& 拼接
                          ↓
              HMAC-SHA256(stringToSign, apiSecret)
                          ↓
                    签名结果（小写 hex）
```

## 3. 创建支付订单

### 3.1 请求信息

| 项目 | 值 |
|------|-----|
| URL | `POST /api/v1/payments` |
| 完整地址 | `https://payment.aieducenter.com/api/v1/payments` |
| Content-Type | `application/json` |
| 签名验证 | **需要** |

### 3.2 请求头

```
X-App-Id: {您的API Key}
X-Timestamp: {当前Unix时间戳}
X-Nonce: {随机唯一字符串}
X-Body-Digest: {请求体SHA-256摘要}
X-Sign: {签名}
Content-Type: application/json
```

### 3.3 请求体

```json
{
  "businessOrderNo": "ORDER_20260410_001",
  "amount": 100,
  "subject": "课程购买-高等数学",
  "body": "高等数学第三章全套课程",
  "businessName": "在线教育平台",
  "notifyUrl": "https://your-business.com/api/payment/callback",
  "expiredSeconds": 3600,
  "attach": "{\"userId\":12345}"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `businessOrderNo` | String | **是** | 您的业务订单号，最长 64 字符，需在您的业务系统内唯一 |
| `amount` | Long | **是** | 支付金额，单位：**分**。如 `100` 表示 1 元，`19900` 表示 199 元 |
| `subject` | String | **是** | 支付标题，最长 255 字符，展示在支付页面上 |
| `body` | String | 否 | 支付描述，最长 1000 字符 |
| `businessName` | String | 否 | 业务名称，最长 128 字符 |
| `notifyUrl` | String | **是** | 支付结果异步通知地址，最长 512 字符，需为 HTTPS 地址 |
| `expiredSeconds` | Long | 否 | 订单过期时间（秒），默认 3600（1 小时） |
| `attach` | String | 否 | 附加数据，最长 1000 字符，原样返回，可用于传递自定义参数 |

### 3.4 响应体

```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "id": 1001,
    "businessOrderNo": "ORDER_20260410_001",
    "paymentOrderNo": "PAY20260410143000001",
    "businessSystemName": "在线教育平台",
    "businessName": "在线教育平台",
    "status": 1,
    "statusName": "待支付",
    "amount": 100,
    "subject": "课程购买-高等数学",
    "body": "高等数学第三章全套课程",
    "paymentChannelName": "工商银行",
    "qrCodeUrl": "https://gw.open.icbc.com.cn/...",
    "clientIp": "192.168.1.100",
    "createdAt": "2026-04-10T14:30:00",
    "expiredAt": "2026-04-10T15:30:00",
    "paidAt": null,
    "bankOrderNo": null,
    "thirdPartyOrderNo": null
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Long | 支付订单内部 ID |
| `businessOrderNo` | String | 您传入的业务订单号 |
| `paymentOrderNo` | String | **支付服务生成的订单号**（后续查询、退款使用此号） |
| `businessSystemName` | String | 业务系统名称 |
| `businessName` | String | 业务名称 |
| `status` | Integer | 订单状态码（见[第 6 节](#6-支付状态说明)） |
| `statusName` | String | 订单状态名称（如"待支付"、"已支付"） |
| `amount` | Long | 支付金额（分） |
| `subject` | String | 支付标题 |
| `body` | String | 支付描述 |
| `paymentChannelName` | String | 支付渠道名称 |
| `qrCodeUrl` | String | **支付二维码链接**，引导用户扫码支付 |
| `clientIp` | String | 客户端 IP |
| `createdAt` | String | 创建时间 |
| `expiredAt` | String | 过期时间 |
| `paidAt` | String | 支付完成时间（未支付时为 null） |
| `bankOrderNo` | String | 银行订单号（支付完成后有值） |
| `thirdPartyOrderNo` | String | 第三方订单号（支付完成后有值） |

### 3.5 接入要点

1. 创建订单成功后，响应中的 `qrCodeUrl` 是支付二维码链接，需引导用户打开或扫码完成支付
2. 请保存 `paymentOrderNo`，后续查询支付状态和发起退款时需要使用
3. `notifyUrl` 为异步回调地址，当支付状态变更时支付服务会向您发送通知（建议同时使用主动查询作为兜底）

## 4. 创建预支付订单

预支付接口适用于**聚合支付**场景（微信、支付宝、云闪付），与普通创建订单不同，该接口会返回前端支付 SDK 所需的 `prepayDataPackage` 参数包。

### 4.1 请求信息

| 项目 | 值 |
|------|-----|
| URL | `POST /api/v1/payments/prepay` |
| 完整地址 | `https://payment.aieducenter.com/api/v1/payments/prepay` |
| Content-Type | `application/json` |
| 签名验证 | **需要** |

### 4.2 请求头

```
X-App-Id: {您的API Key}
X-Timestamp: {当前Unix时间戳}
X-Nonce: {随机唯一字符串}
X-Body-Digest: {请求体SHA-256摘要}
X-Sign: {签名}
Content-Type: application/json
```

### 4.3 请求体

```json
{
  "businessOrderNo": "ORDER_20260410_001",
  "amount": 100,
  "subject": "课程购买-高等数学",
  "body": "高等数学第三章全套课程",
  "businessName": "在线教育平台",
  "notifyUrl": "https://your-business.com/api/payment/callback",
  "expiredSeconds": 3600,
  "attach": "",
  "payMode": 9,
  "accessType": 9,
  "openId": "o1234567890abcdef"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `businessOrderNo` | String | **是** | 您的业务订单号，最长 64 字符，需在您的业务系统内唯一 |
| `amount` | Long | **是** | 支付金额，单位：**分**。如 `100` 表示 1 元 |
| `subject` | String | **是** | 支付标题，最长 255 字符，展示在支付页面上 |
| `body` | String | 否 | 支付描述，最长 1000 字符 |
| `businessName` | String | 否 | 业务名称，最长 128 字符 |
| `notifyUrl` | String | **是** | 支付结果异步通知地址，最长 512 字符，需为 HTTPS 地址 |
| `expiredSeconds` | Long | 否 | 订单过期时间（秒），默认 3600（1 小时） |
| `attach` | String | 否 | 附加数据，最长 1000 字符，原样返回 |
| `payMode` | Integer | **是** | 支付方式（见下方支付方式说明） |
| `accessType` | Integer | **是** | 接入方式（见下方接入方式说明） |
| `openId` | String | 条件必填 | 微信用户标识，最长 128 字符（条件见下方说明） |
| `unionId` | String | 条件必填 | 支付宝用户标识，最长 128 字符（条件见下方说明） |

### 4.4 支付方式（payMode）

| 值 | 说明 |
|----|------|
| `9` | 微信 |
| `10` | 支付宝 |
| `13` | 云闪付 |

### 4.5 接入方式（accessType）

| 值 | 说明 |
|----|------|
| `4` | H5 |
| `5` | APP |
| `7` | 微信公众号 |
| `8` | 支付宝生活号 |
| `9` | 小程序 |

### 4.6 openId / unionId 条件必填说明

| 支付方式 | 接入方式 | 必填字段 |
|----------|----------|----------|
| 微信（`9`） | 微信公众号（`7`） | `openId` |
| 微信（`9`） | 小程序（`9`） | `openId` |
| 支付宝（`10`） | 支付宝生活号（`8`） | `unionId` |
| 其他组合 | — | 均非必填 |

### 4.7 响应体

```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "id": 1002,
    "businessOrderNo": "ORDER_20260410_002",
    "paymentOrderNo": "PAY20260410143000002",
    "businessSystemName": "在线教育平台",
    "status": 1,
    "statusName": "待支付",
    "amount": 100,
    "prepayDataPackage": "{\"appId\":\"wx1234567890\",\"timeStamp\":\"1712736000\",\"nonceStr\":\"abc123\",\"package\":\"prepay_id=wx20260410\",\"signType\":\"RSA\",\"paySign\":\"xxx\"}",
    "payMode": 9,
    "payModeName": "微信",
    "accessType": 9,
    "accessTypeName": "小程序",
    "expiredAt": "2026-04-10T15:30:00",
    "createdAt": "2026-04-10T14:30:00"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Long | 支付订单内部 ID |
| `businessOrderNo` | String | 您传入的业务订单号 |
| `paymentOrderNo` | String | **支付服务生成的订单号**（后续查询、退款使用此号） |
| `businessSystemName` | String | 业务系统名称 |
| `status` | Integer | 订单状态码（见[第 6 节](#6-支付状态说明)） |
| `statusName` | String | 订单状态名称 |
| `amount` | Long | 支付金额（分） |
| `prepayDataPackage` | String | **前端调起支付所需的参数包 JSON**，直接传给对应支付 SDK 即可 |
| `payMode` | Integer | 支付方式码 |
| `payModeName` | String | 支付方式名称 |
| `accessType` | Integer | 接入方式码 |
| `accessTypeName` | String | 接入方式名称 |
| `expiredAt` | String | 过期时间 |
| `createdAt` | String | 创建时间 |

### 4.8 接入要点

1. `prepayDataPackage` 是 JSON 字符串，前端需解析后传给微信/支付宝/云闪付 SDK 发起支付
2. 请保存 `paymentOrderNo`，后续查询支付状态和发起退款时需要使用
3. 回调通知机制与普通支付订单一致（见[第 10 节](#10-异步回调通知)）
4. `payMode` 和 `accessType` 的组合决定了支付渠道和前端接入方式，请确保组合正确

## 5. 主动查询支付状态

### 5.1 请求信息

| 项目 | 值 |
|------|-----|
| URL | `POST /api/v1/payments/{paymentOrderNo}/query` |
| 完整地址 | `https://payment.aieducenter.com/api/v1/payments/{paymentOrderNo}/query` |
| Content-Type | `application/json` |
| 签名验证 | **需要** |

> **注意**：该接口会**实时调用银行网关**查询最新支付状态并更新本地订单，返回的是最新状态。

### 5.2 路径参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `paymentOrderNo` | String | 创建订单时返回的支付订单号 |

### 5.3 请求头

```
X-App-Id: {您的API Key}
X-Timestamp: {当前Unix时间戳}
X-Nonce: {随机唯一字符串}
X-Body-Digest: {SHA-256("")摘要}   ← 无请求体，使用空字符串的摘要
X-Sign: {签名}
Content-Type: application/json
```

> **注意**：该接口无请求体（body 为空），`X-Body-Digest` 应传入空字符串的 SHA-256 值：`e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855`

### 5.4 请求体

无。

### 5.5 响应体

```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "id": 1001,
    "businessOrderNo": "ORDER_20260410_001",
    "paymentOrderNo": "PAY20260410143000001",
    "businessSystemName": "在线教育平台",
    "businessName": "在线教育平台",
    "status": 2,
    "statusName": "已支付",
    "amount": 100,
    "subject": "课程购买-高等数学",
    "body": "高等数学第三章全套课程",
    "paymentChannelName": "工商银行",
    "qrCodeUrl": "https://gw.open.icbc.com.cn/...",
    "clientIp": "192.168.1.100",
    "createdAt": "2026-04-10T14:30:00",
    "expiredAt": "2026-04-10T15:30:00",
    "paidAt": "2026-04-10T14:35:22",
    "bankOrderNo": "ICBC20260410143000001",
    "thirdPartyOrderNo": "TP2026041000001"
  }
}
```

响应字段与创建订单相同（见 [3.4 节](#34-响应体)），重点关注 `status` 和 `statusName` 字段的变化。

### 5.6 接入要点

1. 该接口会**实时查询银行**，建议在用户支付后的适当时机调用（如用户点击"已完成支付"按钮时）
2. 不建议高频轮询，建议间隔 **5-10 秒**以上
3. 当 `status` 为终态（`2`/`3`/`4`/`5`）时，无需再查询

## 6. 支付状态说明

### 6.1 支付订单状态

| 状态码 (`status`) | 状态名称 (`statusName`) | 是否终态 | 说明 |
|-------------------|------------------------|----------|------|
| `1` | 待支付 | 否 | 订单已创建，等待用户扫码支付 |
| `2` | 已支付 | **是** | 用户已完成支付 |
| `3` | 支付失败 | **是** | 支付过程中发生失败 |
| `4` | 已取消 | **是** | 订单已被手动取消 |
| `5` | 已过期 | **是** | 订单超时未支付，自动过期 |

**状态流转**：

```
1(待支付) ──→ 2(已支付)     （用户支付成功）
         ├──→ 3(支付失败)   （支付失败）
         ├──→ 4(已取消)     （主动取消）
         └──→ 5(已过期)     （超时自动过期）
```

### 6.2 建议的判断逻辑

```python
# 判断支付是否成功
if status == 2:
    # 支付成功，处理业务逻辑
    pass
elif status in (3, 4, 5):
    # 终态但非成功，关闭订单
    pass
else:
    # status == 1，仍在支付中
    pass
```

## 7. 创建退款订单

### 7.1 请求信息

| 项目 | 值 |
|------|-----|
| URL | `POST /api/v1/refunds` |
| 完整地址 | `https://payment.aieducenter.com/api/v1/refunds` |
| Content-Type | `application/json` |
| 签名验证 | **需要** |

### 7.2 请求体

```json
{
  "businessOrderNo": "ORDER_20260410_001",
  "paymentOrderNo": "PAY20260410143000001",
  "refundAmount": 100,
  "reason": "课程取消",
  "attach": "",
  "notifyUrl": "https://your-business.com/api/refund/callback",
  "needAudit": false
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `businessOrderNo` | String | **是** | 您的业务订单号，最长 64 字符 |
| `paymentOrderNo` | String | **是** | 原支付订单号（创建支付订单时返回的 `paymentOrderNo`） |
| `refundAmount` | Long | **是** | 退款金额，单位：**分**，不能超过原支付金额 |
| `reason` | String | 否 | 退款原因，最长 512 字符 |
| `attach` | String | 否 | 附加数据，最长 1000 字符，原样返回 |
| `notifyUrl` | String | 否 | 退款结果异步通知地址，最长 512 字符，需为 HTTPS 地址。不传则不回调 |
| `needAudit` | Boolean | 否 | 是否需要支付服务审核。`false`：免审，直接发起退款；`true` 或不传：需人工审核 |

### 7.3 响应体

```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "id": 2001,
```

### 7.4 免审退款说明

当 `needAudit` 传 `false` 时，支付服务会自动审核通过并立即发起银行退款。此时响应中的状态可能直接为 `4`（退款中）、`5`（退款成功）或 `6`（退款失败），而不是 `1`（待审核）。

### 7.5 接入要点

1. 请确保原支付订单已支付成功（`status` = 2）才能发起退款
2. 退款金额不能超过原支付金额
3. 建议保存 `refundOrderNo`，后续查询退款状态时需要使用
4. 如果需要快速退款（无需人工审核），设置 `needAudit` 为 `false`

## 9. 退款状态说明

### 9.1 退款订单状态

| 状态码 (`status`) | 状态名称 (`statusName`) | 是否终态 | 说明 |
|-------------------|------------------------|----------|------|
| `1` | 待审核 | 否 | 退款订单已创建，等待审核（`needAudit=true` 时） |
| `2` | 已拒绝 | **是** | 审核未通过 |
| `3` | 已批准 | 否 | 审核通过，等待发起银行退款 |
| `4` | 退款中 | 否 | 已向银行发起退款请求，等待银行处理 |
| `5` | 退款成功 | **是** | 退款到账 |
| `6` | 退款失败 | **是** | 退款失败 |

**状态流转**：

```
1(待审核) ──→ 2(已拒绝)     （审核拒绝，会回调业务端）
          └──→ 3(已批准)     （审核通过）
                   └──→ 4(退款中)   （已向银行发起退款）
                            ├──→ 5(退款成功) （退款到账，会回调业务端）
                            └──→ 6(退款失败) （退款失败，会回调业务端）
```

**免审模式（`needAudit=false`）**：跳过人工审核，自动从 1(待审核) → 3(已批准) → 4(退款中)。

### 9.2 建议的判断逻辑

```python
if status == 5:
    # 退款成功，处理业务逻辑（如恢复库存、关闭订单等）
    pass
elif status == 6:
    # 退款失败，可根据业务需要重试或人工处理
    pass
elif status == 2:
    # 审核拒绝，通知用户或做其他处理
    pass
else:
    # 1/3/4 仍在处理中，等待回调或轮询
    pass
```

## 10. 异步回调通知

当支付或退款状态变为终态时，支付服务会向您在创建订单时提供的 `notifyUrl` 发送 **POST** 请求，通知结果。

### 10.1 通知机制说明

| 项目 | 说明 |
|------|------|
| 触发时机 | 银行回调通知支付服务后，支付服务立即异步通知业务系统 |
| 请求方式 | `POST` |
| 数据格式 | `application/json` |
| 超时时间 | 10 秒 |
| 重试机制 | **不重试**。如果通知失败，业务系统可通过主动查询接口获取结果（[支付查询](#5-主动查询支付状态)、[退款查询](#8-查询退款状态)） |
| 幂等处理 | 同一订单可能收到多次通知（如银行重复回调），业务系统需做幂等处理 |

### 10.2 支付回调请求体

支付回调请求体与创建订单/查询状态的响应体结构**完全一致**（`PaymentOrderResponse`），业务系统无需为回调定义额外的数据结构。

### 10.4 业务系统响应要求

业务系统收到通知后，应返回 **HTTP 200** 状态码表示接收成功。支付服务根据 HTTP 状态码判断通知是否成功：

| 响应 | 含义 |
|------|------|
| HTTP 200 | 通知成功 |
| 非 200 / 超时 / 网络异常 | 通知失败，服务端仅记录日志，**不会重试** |

### 10.5 接入建议

1. **幂等处理**：根据订单号去重（支付用 `paymentOrderNo`，退款用 `refundOrderNo`），同一笔订单多次通知不应重复处理
2. **先查再改**：收到通知后建议先通过 `businessOrderNo` 查询本地订单状态，确认订单仍为待处理再更新
3. **主动查询兜底**：由于回调不重试，建议业务系统在关键节点主动调用查询接口确认最终状态
4. **异步处理**：回调处理逻辑应尽量快速返回 200，耗时操作建议异步处理

## 11. 错误处理

### 11.1 响应格式

所有错误响应遵循统一格式：

```json
{
  "code": 401,
  "message": "签名不匹配",
  "data": null
}
```

### 11.2 常见错误码

#### 签名相关错误

| 错误码 | HTTP 状态码 | 说明 | 处理建议 |
|--------|-------------|------|----------|
| `OAS_002` | 400 | 签名验证失败（缺少必要请求头） | 检查是否携带了全部 5 个自定义请求头 |
| `OAS_040` | 400 | 时间戳已过期 | 检查服务器时间是否准确，确保时间戳在 300 秒窗口内 |
| `OAS_042` | 400 | Nonce 重复使用 | 每个 Nonce 只能使用一次，建议使用 UUID |
| `OAS_043` | 400 | Body Digest 不匹配 | 检查请求体摘要计算是否正确 |
| `OAS_045` | 401 | 签名不匹配 | 检查签名计算流程，确保参数正确、排序正确 |
| `OAS_010` | 404 | API Key 不存在 | 检查 X-App-Id 是否正确 |
| `OAS_020` | 403 | API Key 已禁用 | 联系管理员确认 Key 状态 |

#### 业务相关错误

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| `PAY_001` | 400 | 支付订单号格式不正确 |
| `PAY_030` | 404 | 支付订单不存在 |
| `PAY_040` | 403 | 支付订单已过期 |
| `PAY_050` | 409 | 业务订单号重复 |

#### 退款相关错误

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| `PAY_035` | 404 | 退款订单不存在 |
| `PAY_031` | 403 | 原支付订单未支付成功 |
| `PAY_032` | 400 | 退款金额超过可退款金额 |
| `PAY_033` | 400 | 退款订单不是待审核状态 |
| `PAY_034` | 400 | 退款订单不是已批准状态 |

### 11.3 HTTP 状态码说明

| 状态码 | 含义 |
|--------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误或签名校验失败 |
| 401 | 签名不匹配 |
| 403 | 资源不可访问（如订单已过期、Key 已禁用） |
| 404 | 资源不存在 |
| 409 | 资源冲突（如订单号重复） |

以下提供多种语言的签名计算和接口调用示例。

### 12.1 Java 示例

```java
import javax.crypto.Mac;
import javax.crypto.spec.SecretKeySpec;
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.nio.charset.StandardCharsets;
import java.security.MessageDigest;
import java.util.HexFormat;
import java.util.Map;
import java.util.TreeMap;
import java.util.UUID;

public class PaymentClient {

    private static final String BASE_URL = "https://payment.aieducenter.com";
    private static final String API_KEY = "your_api_key";
    private static final String API_SECRET = "your_api_secret";

    private final HttpClient httpClient = HttpClient.newHttpClient();

    // ==================== 创建支付订单 ====================

    public void createPayment() throws Exception {
        String body = """
            {
              "businessOrderNo": "ORDER_20260410_001",
              "amount": 100,
              "subject": "课程购买-高等数学",
              "body": "高等数学第三章全套课程",

              "notifyUrl": "https://your-business.com/api/payment/callback",
              "expiredSeconds": 3600,
              "attach": ""
            }
            """;

        String response = sendRequest("POST", "/api/v1/payments", body);
        System.out.println("创建订单响应: " + response);
    }

    // ==================== 主动查询支付状态 ====================

    public void queryPaymentStatus(String paymentOrderNo) throws Exception {
        String path = "/api/v1/payments/" + paymentOrderNo + "/query";
        // 无请求体
        String response = sendRequest("POST", path, "");
        System.out.println("查询状态响应: " + response);
    }

    // ==================== 创建退款订单 ====================

    public void createRefund() throws Exception {
        String body = """
            {
              "businessOrderNo": "ORDER_20260410_001",
              "paymentOrderNo": "PAY20260410143000001",
              "refundAmount": 100,
              "reason": "课程取消",
              "attach": "",
              "notifyUrl": "https://your-business.com/api/refund/callback",
              "needAudit": false
            }
            """;

        String response = sendRequest("POST", "/api/v1/refunds", body);
        System.out.println("创建退款响应: " + response);
    }

    // ==================== 查询退款状态 ====================

    public void queryRefundStatus(String refundOrderNo) throws Exception {
        String path = "/api/v1/refunds/" + refundOrderNo;
        // GET 请求，无请求体
        String response = sendGetRequest(path);
        System.out.println("查询退款响应: " + response);
    }

    // ==================== 通用请求方法 ====================

    private String sendGetRequest(String path) throws Exception {
        String timestamp = String.valueOf(System.currentTimeMillis() / 1000);
        String nonce = UUID.randomUUID().toString();
        String bodyDigest = sha256Hex("");  // GET 无请求体

        TreeMap<String, String> signParams = new TreeMap<>();
        signParams.put("appId", API_KEY);
        signParams.put("bodyDigest", bodyDigest);
        signParams.put("nonce", nonce);
        signParams.put("timestamp", timestamp);

        String stringToSign = joinParams(signParams);
        String signature = hmacSha256(stringToSign, API_SECRET);

        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create(BASE_URL + path))
                .GET()
                .header("Content-Type", "application/json")
                .header("X-App-Id", API_KEY)
                .header("X-Timestamp", timestamp)
                .header("X-Nonce", nonce)
                .header("X-Body-Digest", bodyDigest)
                .header("X-Sign", signature)
                .build();

        HttpResponse<String> response = httpClient.send(request, HttpResponse.BodyHandlers.ofString());
        return response.body();
    }

    private String sendRequest(String method, String path, String body) throws Exception {
        String timestamp = String.valueOf(System.currentTimeMillis() / 1000);
        String nonce = UUID.randomUUID().toString();
        String bodyDigest = sha256Hex(body);

        // 构造待签名字符串（按键名字典序排列）
        TreeMap<String, String> signParams = new TreeMap<>();
        signParams.put("appId", API_KEY);
        signParams.put("bodyDigest", bodyDigest);
        signParams.put("nonce", nonce);
        signParams.put("timestamp", timestamp);

        String stringToSign = joinParams(signParams);
        String signature = hmacSha256(stringToSign, API_SECRET);

        // 构造请求
        HttpRequest.Builder requestBuilder = HttpRequest.newBuilder()
                .uri(URI.create(BASE_URL + path))
                .header("Content-Type", "application/json")
                .header("X-App-Id", API_KEY)
                .header("X-Timestamp", timestamp)
                .header("X-Nonce", nonce)
                .header("X-Body-Digest", bodyDigest)
                .header("X-Sign", signature);

        if ("POST".equals(method)) {
            requestBuilder.POST(HttpRequest.BodyPublishers.ofString(body));
        }

        HttpResponse<String> response = httpClient.send(
                requestBuilder.build(),
                HttpResponse.BodyHandlers.ofString()
        );

        return response.body();
    }

    // ==================== 工具方法 ====================

    /** 计算 SHA-256 并转为小写十六进制 */
    private static String sha256Hex(String data) throws Exception {
        MessageDigest digest = MessageDigest.getInstance("SHA-256");
        byte[] hash = digest.digest(data.getBytes(StandardCharsets.UTF_8));
        return HexFormat.of().formatHex(hash);
    }

    /** 计算 HMAC-SHA256 并转为小写十六进制 */
    private static String hmacSha256(String data, String secret) throws Exception {
        Mac mac = Mac.getInstance("HmacSHA256");
        SecretKeySpec keySpec = new SecretKeySpec(
                secret.getBytes(StandardCharsets.UTF_8), "HmacSHA256");
        mac.init(keySpec);
        byte[] hash = mac.doFinal(data.getBytes(StandardCharsets.UTF_8));
        return HexFormat.of().formatHex(hash);
    }

    /** 按 key=value&key=value 拼接 */
    private static String joinParams(TreeMap<String, String> params) {
        StringBuilder sb = new StringBuilder();
        for (Map.Entry<String, String> entry : params.entrySet()) {
            if (!sb.isEmpty()) sb.append("&");
            sb.append(entry.getKey()).append("=").append(entry.getValue());
        }
        return sb.toString();
    }
}
```

## 附录：签名验证自测清单

接入完成后，请按以下清单逐项排查签名问题：

- [ ] 请求头 `X-App-Id` / `X-Timestamp` / `X-Nonce` / `X-Body-Digest` / `X-Sign` 五个头部是否齐全
- [ ] `X-Timestamp` 是否为当前 Unix 时间戳（秒级），且与服务端时间差在 300 秒以内
- [ ] `X-Nonce` 是否每次请求都唯一（建议使用 UUID）
- [ ] `X-Body-Digest` 是否为请求体原始字节的 SHA-256（小写 hex）
- [ ] 待签名字符串中参数是否按**键名字典序**排列
- [ ] HMAC-SHA256 的密钥是 `apiSecret`，不是 `apiKey`
- [ ] HMAC-SHA256 结果是否为**小写**十六进制
- [ ] GET/POST 无请求体时，Body Digest 是否使用了空字符串的 SHA-256 值
